## What & Why

This is the frontend draft editor for invoices — the main UI a user interacts with to create and edit invoice drafts. It was originally scoped as T07 PR3 but split into its own ticket because the frontend is a standalone deliverable with its own test surface.

A user visits `/business/invoices/new` → a draft is created automatically → they're redirected to the edit page where they fill in customer, line items, dates, notes → save explicitly with "שמור טיוטה".

---

## Key Design Decisions

1. **No autosave** — explicit "שמור טיוטה" button only
2. **No finalize button** — T08 scope
3. **3 document types** — tax_invoice, tax_invoice_receipt, receipt (no credit_note)
4. **Per-line VAT rate** — Select (17%/פטור) per row; forced 0 for receipts and exempt dealers
5. **Currency display** — form uses the display unit, converted to minor units (`Math.round(val * 100)`) on save
6. **`defaultVatRate` from business detail** — fetched via `fetchBusiness()` on the edit page
7. **Navbar stays disabled** — T09 enables it. QuickActions buttons enabled now.
