## What & Why

The SHAAM service layer (T12–T14) is fully wired with a mock client for dev/test and an HTTP client shell for sandbox/production. However, the HTTP client (`api/src/services/shaam/http-client.ts`) currently throws "not implemented" for both methods:

- `requestAllocationNumber()` — needed to get ITA allocation numbers for invoices above threshold
- `reportEmergencyUsage()` — needed to report emergency number usage to ITA

Without this, `SHAAM_MODE=production` will fail at runtime. This ticket implements the real HTTP calls against the ITA SHAAM API.

### Implementation details

1. **Allocation request**: POST XML/JSON payload to ITA endpoint, parse allocation number from response
2. **Emergency usage report**: POST usage data to ITA reporting endpoint
3. **Error handling**: Map ITA error codes to `AppError` instances with meaningful messages
4. **Retry logic**: ITA API can be slow/flaky — implement retry with exponential backoff
5. **Encryption**: Payload must be encrypted with `SHAAM_ENCRYPTION_KEY` per ITA spec

### Out of scope

- Changing the mock client or service abstraction (already done in T12)
- Token management (already done in T12)
- Job queue wiring (already done in T13)
