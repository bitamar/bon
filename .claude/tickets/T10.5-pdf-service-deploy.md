## What & Why

The `pdf/` workspace is a standalone Fastify + Puppeteer microservice that renders invoices to PDF. The code was complete (T10) but needed its own Railway service so the API can call it in production.

The PDF service is separate from the API because Chromium (~400MB) would bloat the API image and PDF rendering is CPU-intensive — independent scaling is needed.

- The PDF service uses `puppeteer-core` (not `puppeteer`) so Chromium is installed separately in Dockerfile
