## What & Why

PCN874 is ITA's machine-readable format for the monthly/bi-monthly VAT report. VAT-registered businesses (עוסק מורשה, חברה בע"מ) must submit this to the tax authority. BON generates the file — the user downloads it and submits (or hands to their accountant). Not an API submission yet.

---

## PREREQUISITE: Obtain ITA Spec

**This ticket cannot be implemented without the official ITA PCN874 format specification.** The spec defines exact field positions, widths, encoding, and validation rules.

**Action items before implementation:**
1. Request the official "קבצי הנהלת חשבונות ממוחשבת — מבנה PCN874" document from ITA
2. Determine file encoding: UTF-8 or Windows-1255 (ITA sometimes requires the latter)
3. Determine the exact format version currently accepted
4. Obtain access to ITA's PCN874 validation simulator
5. Document findings in this ticket's Architecture Notes section

**Responsible**: Product owner or legal contact. This is an administrative task, not a coding task.

## Acceptance Criteria

- [ ] `GET /businesses/:businessId/reports/pcn874?year=2026&month=02` — download PCN874 file
- [ ] File format strictly per ITA spec: fixed-width or CSV per "קבצי הנהלת חשבונות ממוחשבת" spec
- [ ] Per-record fields: document type code, date, invoice number, customer VAT number, total amount, VAT amount, allocation number (if any)
- [ ] Only includes finalized invoices (drafts excluded)
- [ ] Credit notes included with document type code 330
- [ ] Bi-monthly reporting: if business reports bi-monthly, the endpoint accepts `month=01` for Jan-Feb period (clarify with accountant)
- [ ] Validated against ITA's own simulator before shipping
- [ ] "הורד דוח מע״מ" button in reporting section of UI
  - [ ] Month/year picker (`Select` for month, `NumberInput` for year)
  - [ ] Download triggers file download (same blob pattern as PDF)
- [ ] Applies only to businesses with VAT (עוסק פטור excluded — they don't have VAT)
- [ ] `npm run check` passes
