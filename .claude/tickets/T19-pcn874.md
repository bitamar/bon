## What & Why

PCN874 is ITA's machine-readable format for the monthly/bi-monthly VAT report. VAT-registered businesses (עוסק מורשה, חברה בע"מ) must submit this to the tax authority. BON generates the file — the user downloads it and submits (or hands to their accountant). Not an API submission yet.

---

## Acceptance Criteria

### API
- [ ] `GET /businesses/:businessId/reports/pcn874?year=2026&month=02` — returns PCN874 file download
- [ ] Query params validated with Zod: `year` (4-digit number), `month` (1-12)
- [ ] Response: `Content-Type: text/plain; charset=windows-1255`, `Content-Disposition: attachment; filename="PCN874_{vatNumber}_{YYYYMM}.txt"`
- [ ] Returns 422 if business is `exempt_dealer` (עוסק פטור doesn't report VAT)
- [ ] Returns 200 with a valid file even if no invoices exist for the period (opening + closing records, zero totals)
- [ ] Only includes **non-draft** invoices (status: `finalized`, `sent`, `paid`, `partially_paid`, `cancelled`, `credited`)
- [ ] Filters invoices by `invoiceDate` (the tax point date / מועד החיוב במס), not `issuedAt`
- [ ] All amounts in **agorot** (minor units) as stored in DB, formatted as zero-padded integers per field width

### PCN874 File Format
- [ ] **Encoding**: Windows-1255 (ITA requirement for Hebrew text compatibility)
- [ ] **Line terminator**: CRLF (`\r\n`)
- [ ] **Three record types**: Opening (O), Detail (S for sales), Closing (X)
- [ ] File format is **fixed-width text** per ITA spec

### Opening Record (רשומת פתיחה) — Record Type "O"

| # | Field | Type | Width | Description |
|---|-------|------|-------|-------------|
| 1 | סוג רשומה | A | 1 | Fixed: `O` |
| 2 | מספר עוסק מורשה | N | 9 | Business VAT registration number, zero-padded |
| 3 | תקופת דיווח | N | 6 | Reporting period `YYYYMM` |
| 4 | סוג דוח | N | 1 | Fixed: `1` (regular report) |
| 5 | תאריך יצירת קובץ | N | 8 | File creation date `YYYYMMDD` |
| 6 | סימן סכום עסקאות חייבות | A | 1 | `+` or `-` |
| 7 | סכום עסקאות חייבות | N | 11 | Total taxable transactions (excl. VAT), zero-padded |
| 8 | סימן מע"מ עסקאות חייבות | A | 1 | `+` or `-` |
| 9 | מע"מ עסקאות חייבות | N | 9 | VAT on taxable transactions, zero-padded |
| 10 | סימן עסקאות פטורות/אפס | A | 1 | `+` or `-` |
| 11 | סכום עסקאות פטורות/אפס | N | 11 | Total zero-rate/exempt transactions, zero-padded |
| 12 | סימן תשומות | A | 1 | `+` (BON is sales-only, always `+`) |
| 13 | סכום תשומות | N | 11 | `00000000000` (BON doesn't track inputs) |
| 14 | סימן מע"מ תשומות | A | 1 | `+` |
| 15 | מע"מ תשומות | N | 9 | `000000000` |
| 16 | מספר רשומות | N | 9 | Count of detail records in file |

### Detail Record (רשומת תנועה) — Record Type "S" (Sales)

| # | Field | Type | Width | Description |
|---|-------|------|-------|-------------|
| 1 | סוג רשומה | A | 1 | `S` (sales transaction) |
| 2 | סוג תנועה | N | 2 | Entry type code (see mapping below) |
| 3 | מספר עוסק צד שני | N | 9 | Counterparty VAT number (customer `taxId`), `000000000` if missing |
| 4 | מספר חשבונית | N | 9 | Invoice `sequenceNumber`, zero-padded (right-aligned) |
| 5 | תאריך חשבונית | N | 8 | `YYYYMMDD` from `invoiceDate` |
| 6 | סימן סכום | A | 1 | `+` or `-` (credit notes = `-`) |
| 7 | סכום לפני מע"מ | N | 11 | `totalExclVatMinorUnits`, zero-padded |
| 8 | סימן מע"מ | A | 1 | `+` or `-` (matches sign of amount) |
| 9 | סכום מע"מ | N | 9 | `vatMinorUnits`, zero-padded |
| 10 | מספר הקצאה | N | 9 | SHAAM allocation number (right 9 digits), `000000000` if none |

### Entry Type Code Mapping (סוג תנועה)

BON is a **sales-only** platform (no purchase/input tracking). All records use the "S" record type with these entry type codes:

| BON Document Type | Entry Type Code | Description |
|---|---|---|
| `tax_invoice` | `01` | חשבונית מס — Regular tax invoice |
| `tax_invoice_receipt` | `02` | חשבונית מס קבלה — Tax invoice receipt |
| `receipt` | `03` | קבלה — Receipt (zero VAT) |
| `credit_note` | `11` | חשבונית מס זיכוי — Credit note (negative amounts) |

### Closing Record (רשומת סגירה) — Record Type "X"

| # | Field | Type | Width | Description |
|---|-------|------|-------|-------------|
| 1 | סוג רשומה | A | 1 | Fixed: `X` |
| 2 | מספר רשומות | N | 9 | Same count as opening record field 16 |

### Sign & Amount Rules

- **Regular invoices** (`tax_invoice`, `tax_invoice_receipt`, `receipt`): sign = `+`, amounts are positive
- **Credit notes**: sign = `-`, amounts are the absolute value of the credit (stored as positive in DB, negated in PCN874)
- **Cancelled invoices**: **excluded** from PCN874 (they should have a corresponding credit note if they need to be reversed)
- **Zero-VAT transactions** (receipts, VAT-exempt): counted in field 11 (exempt/zero-rate total), not field 7 (taxable total)
- Amounts are in **agorot** (minor units) — the DB already stores them this way

### Allocation Number Handling

| Allocation Status | PCN874 Value |
|---|---|
| `approved` | Right 9 digits of `allocationNumber` |
| `emergency` | Right 9 digits of `allocationNumber` |
| `pending` | `000000000` (not yet received) |
| `rejected` | `000000000` |
| `null` | `000000000` (below threshold, not required) |

## Scope Limitations (explicit non-goals)

- **No input/purchase tracking**: BON is sales-only. Input fields (fields 12-15 in opening record) are always zero.
- **No bi-monthly support**: MVP uses monthly periods only. Bi-monthly businesses can download two months separately. (Future: T19.1)
- **No ITA submission**: User downloads the file and submits manually or via accountant. (Future: direct API submission)
- **No file validation against ITA simulator**: Deferred to post-MVP. File format correctness is verified by unit tests.
- **No petty cash aggregation**: Not applicable — BON tracks individual invoices.

## Architecture Notes

### API Layer

- **Route**: `api/src/routes/pcn874.ts` — registers `GET /businesses/:businessId/reports/pcn874`
- **Service**: `api/src/services/pcn874-service.ts` — `generatePcn874(businessId, year, month): Buffer`
- **Repository**: Reuse `invoice-repository.findInvoices()` with `dateFrom`/`dateTo` filter + status exclusion of `draft` and `cancelled`
- **Zod schemas**: Add `pcn874QuerySchema` in `types/src/pcn874.ts` for `{ year, month }`
- **Register route** in `api/src/app.ts`

### Service Logic

1. Validate business is not `exempt_dealer`
2. Query invoices for the period: `invoiceDate >= YYYY-MM-01` AND `invoiceDate <= YYYY-MM-lastDay`, status NOT IN (`draft`, `cancelled`)
3. Build opening record: sum taxable amounts, VAT, zero-rate amounts
4. Build detail records: one per invoice, sorted by `invoiceDate` then `sequenceNumber`
5. Build closing record
6. Encode to Windows-1255 buffer

### Encoding

Use `iconv-lite` (already common in Node.js) to convert UTF-8 strings to Windows-1255 buffer. Add as a dependency if not present.

### Frontend

- **Page**: `front/src/pages/VatReport.tsx`
- **API hook**: `front/src/pages/vat-report/api.ts` — uses `fetch` with `blob()` response for file download
- **Route registration**: Add to `front/src/App.tsx` router

## Open Questions (for Architect to resolve before implementation)

1. **Exact entry type codes**: The codes 01/02/03/11 above are provisional. The Architect must verify against the ITA specification PDF. The official ITA spec is at `kb.sye.co.il/Files/Book1/ISR_/ISR_PCN874.pdf` (may need manual download).
2. **Receipt inclusion**: Should receipts (type `receipt`, no VAT) be included in PCN874 at all? They have zero VAT and are for non-VAT transactions. The Architect should verify.
3. **Windows-1255 necessity**: Modern ITA systems may accept UTF-8. The Architect should verify whether Windows-1255 is still required or if UTF-8 is accepted.
